闭合和注释（判断是否可以进行注入）

闭合：

数字型不需，字符型和搜索型需要

注释:

判断是否可以注入：引号大法（加单引号或注释#’）（一般用于字符型）和and大法（对比发现）（一般用于数字型）

判断注入类型：

总的分为两大类：字符型和数字型。（检验方法详见：https://blog.csdn.net/ewyherayh/article/details/106524972）

搜索型（一般很少用到，依旧详见）

整型注入

https://blog.csdn.net/weixin_45840241/article/details/139106947

当输入的参数是整型时，为数字型注入

测试步骤：

  • 加单引号 URL：xxx.xxx.xxx/xxx.php?id=3’; 对应的SQL:select*from table where id=3’ 此时SQL语句出错，程序无法正常从数据库中查询出数据，就会抛出异常
  • 加and1=1 URL:xxx.xxx.xxx/xxx.php?id=3 and 1=1; 对应的SQL：select*from table where id=3’ and 1=1语句执行正常，与原始页面没有差异
  • 加and1=2 URL:xxx.xxx.xxx/xxx.php？id=3 and 1=2; 对应的SQL：select*from table where id=3 and 1=2语句可以正常执行，但无法查询结果，返回数据与原始页面没有差异

若满足以上三点，可判断URL存在数字型注入

字符注入

当输入的参数是字符串时，为字符注入。与数字型最大的区别在于，字符注入需要单引号来闭合。在构造payload时通过闭合单引号成功执行语句。

数字型 字符型
select*from table where id=3； select * from table where name=’admin’；

测试步骤：

  • 加单引号 URL:select*from table where name=’admin’’; 由于加单引号变成三个单引号，程序无法执行报错
  • 加‘and1=1 SQL：selectfrom table where name=’admin’and 1=1’ 无法进行注入，需要注释符号绕过 构造语句：selectfrom table where name=’admin’and 1=–’ 可成功执行返回结果
  • 加and1=2— SQL: select*from table where name=’admin’ and 1=2—‘ 会报错

若满足以上三点，可判断URL存在字符注入
